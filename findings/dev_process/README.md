# Dev Process Findings

This directory records durable findings about how the thesis skill package is
built, maintained, evaluated, and governed.

Add a finding when a future maintainer or agent would otherwise have to
rediscover an important lesson, such as:

- product-scope or architecture decisions
- stale, incorrect, or misleading documentation
- issue-triage decisions and backlog resets
- data-pipeline constraints or recurring scrape failures
- eval, CI, release, or maintenance-process learnings
- privacy, evidence, or public-data policy decisions
- recurring failure modes that should shape future work

Do not use this directory for temporary scratch notes or routine status updates.
Use `STATUS.md` for current progress, blockers, and dated work logs.

## Filename Convention

Use:

```text
YYYY-MM-DD-short-topic.md
```

Example:

```text
2026-06-25-issue-backlog-reset.md
```

## Entry Shape

Each finding should include:

- Date
- Context
- Finding
- Implication
- Follow-up or linked issue

## Privacy

Never store private student data here. That includes transcripts, CVs, grades,
personal constraints, contact drafts, private advising notes, or inferred
sensitive personal data.
